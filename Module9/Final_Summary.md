The XRISC-32 is meant to be a basic processor with a pipelined data path that prioritizes efficiency and consistent instruction throughput. As shown in a diagram on the final submission file, the pipeline is divided into 5 stages, as well a forwarding method, to manage data hazards and mitigate stalls. The memory hierarchy is designed in a unique way to favor hit rates and reduced AMAT by keeping an optimal block size of 16 bytes and having a 4-way set associativity, so that cache lines can be associated with up to four sets allowing for efficient management and faster memory access.
